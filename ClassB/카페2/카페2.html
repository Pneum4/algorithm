[문제 설명]

N개의 종류의 음료를 제공하는 카페가 있다. 각 음료는 1부터 N까지 고유 번호가 부여되어 있다.

 

손님에게 주문을 받는다.

주문은 여러 개의 음료로 구성되어 있다. 한 주문에 같은 음료가 여러 개 있을 수 있다.

 

어떤 음료가 제조되면 해당 음료가 있는 주문 중 가장 먼저 받은 주문에게 배치되고 주문에서 해당 음료 1개가 삭제된다.

주문에 있는 모든 음료가 제조되면 그 음료들은 손님에게 전달되고 해당 주문은 삭제된다.

만약, 해당 음료가 있는 주문이 없는 경우 그 음료는 바로 버려진다.

 

손님은 주문을 취소할 수 있다.

만약 주문이 취소되면 그 주문에 배치된 음료들은 다른 주문들에게 재배치된다. 재배치되는 과정은 배치되는 과정과 동일하다.

만약 재배치될 수 없는 음료는 바로 버려진다. 취소된 주문은 삭제된다.

 

예로, 주문 1이 음료 1, 1, 3, 4이고 주문 2이 음료 1, 2이고 주문 3이 음료 1, 2, 3인 3개의 주문이 있다고 생각하자. 주문 1이 주문 2보다 먼저 받은 주문이고 주문 2는 주문 3보다 먼저 받은 주문이다.

음료 1이 제조되면 주문 1에 배치된다. 주문 1의 음료 1은 삭제되고 음료 1, 3, 4만 남는다.

음료 2가 제조되면 주문 2에 배치된다. 주문 2의 음료 2는 삭제되고 음료 1만 남는다.

음료 3이 제조되면 주문 1에 배치된다. 주문 1의 음료 3은 삭제되고 주문 1에는 음료 1, 4만 남는다.

음료 4가 제조되면 주문 1에 배치된다. 주문 1의 음료 4는 삭제되고 주문 1에는 음료 1만 남는다.

주문 1이 취소되면 주문 1에 배치된 음료 1은 주문 2, 음료 3은 주문 3에 재배치된다. 주문 1에 배치된 음료 4는 재배치될 수 없으므로 버려진다.

주문 2의 음료 1은 삭제되고 모든 음료가 제작되어 손님에게 전달되고 해당 주문은 삭제된다.

주문 3의 음료 3은 삭제되고 음료 1, 2만 남는다.

 

이미 모든 음료가 제조되어 손님에게 전달된 주문은 취소될 수 없다.

 

또한, 가장 급하게 처리해야 할 주문 최대 5개를 보여준다.

가장 급하게 처리해야 할 주문은 남은 음료의 개수가 가장 많은 주문이다. 그런 주문이 여러 개인 경우 그 중에서 가장 먼저 받은 주문이다.

 

아래 API 설명을 참조하여 각 함수를 구현하라.

※ 아래 함수 signature는 C/C++에 대한 것으로 다른 언어에 대해서는 제공되는 Main과 User Code를 참고하라.

아래는 User Code 부분에 작성해야 하는 API의 설명이다.

void init(int N)

테스트 케이스에 대한 초기화 함수. 각 테스트 케이스의 맨 처음 1회 호출된다.

 

카페에 음료의 종류는 N개가 있다. 각 음료는 1부터 N까지의 수로 구별된다.

 

초기에 어떤 주문도 없다.

 

Parameters

  N : 음료 종류의 개수 (3 ≤ N ≤ 10)

int order(int mID, int M, int mBeverages[])

ID가 mID이고 M개의 음료로 구성된 새로운 주문을 받는다. 주문을 받은 후 남은 주문의 개수를 반환한다.

 

주문을 구성하는 각 음료는 mBeverages[i]로 주어진다. (0 ≤ i ≤ M – 1)

주문 내에 같은 종류의 음료도 있을 수 있다.

 

해당 함수가 먼저 호출되어 생성된 주문이 먼저 받은 주문이다.

 

함수가 호출 시 전달되는 mID 값은 이전 호출에서 이미 전달된 값이 아님을 보장한다.

 

Parameters

  mID : 주문의 ID (1 ≤ mID ≤ 1,000,000,000)

  M : 주문을 구성하는 음료의 개수 (1 ≤ M ≤ 10)

  mBeverages : 주문을 구성하는 음료 (1 ≤ mBeverages[] ≤ N)

 

Return Value

  남은 주문의 개수

int supply(int mBeverage)

음료의 종류가 mBeverage인 음료 1개가 제조되어 공급된다.

 

음료 mBeverage가 있는 주문 중에 가장 먼저 받은 주문에게 해당 음료는 배치된다.

음료가 배치되면 주문에 있는 음료 mBeverage 1개가 삭제된다.

 

해당 음료가 배치에 성공하면 배치된 주문의 ID를 반환한다. 만약 배치에 실패하면 -1을 반환한다.

 

배치에 실패할 경우는 남아 있는 모든 주문들에서 음료 mBeverage가 없는 경우이다. 이런 경우 제조된 음료는 바로 버려지고 다음에 사용되지 않는다.

 

만약 주문에 있는 모두 음료가 제조되어 손님에게 전달된 경우 해당 주문은 삭제된다.

 

Parameters

mBeverage : 완성된 음료 (1 ≤ mBeverage ≤ N)

 

Return Value

음료가 배치된 주문의 ID, 실패하면 -1.

int cancel(int mID)

주문 mID를 취소한다. 취소되기 전 주문에 남은 음료의 개수를 반환한다.

 

주문 mID가 취소가 되면 주문 mID에 배치된 음료들은 재배치된다. 재배치되는 과정은 배치되는 과정과 동일하다.

즉, 그 음료가 있는 가장 먼저 받은 주문에게 재배치되고 해당 주문에서 그 음료 1개가 삭제된다.

 

주문 mID가 이미 모든 음료가 제조되어 손님에게 전달된 경우에는 취소에 실패하고 0을 반환한다.

주문 mID가 이미 취소된 경우에는 취소에 실패하고 -1을 반환한다.

 

주문 mID는 이미 받은 주문임을 보장한다.

 

Parameters

  mID : 취소될 주문의 ID (1 ≤ mID ≤ 1,000,000,000)

 

Return Value

  취소되기 전 남은 음료의 개수, 만약 해당 주문이 완료된 경우 0, 주문이 이미 취소된 경우 -1.

int getStatus(int mID)

주문 mID의 남은 음료의 개수를 반환한다.

 

주문 mID가 모든 음료가 제조되어 손님에게 전달된 경우에는 0을 반환한다.

주문 mID가 취소된 경우에는 -1을 반환한다.

 

주문 mID는 이미 받은 주문임을 보장한다.

 

Parameters

mID : 남은 음료의 개수를 확인할 주문의 ID (1 ≤ mID ≤ 1,000,000,000)

 

Return Value

  주문 mID의 남은 음료의 개수, 만약 해당 주문이 완료된 경우 0, 주문이 취소된 경우 -1.

RESULT hurry()

남은 주문 중 가장 급하게 처리해야 할 주문을 최대 5개 찾아 RESULT 구조체에 저장하여 반환한다.

 

반환할 때 찾은 주문의 개수는 RESULT.cnt에 저장하고 i번째 가장 급하게 처리해야 할 주문의 ID는 RESULT.IDs[i – 1] 저장한다. (1 ≤ i ≤ RESULT.cnt)

 

가장 급하게 처리해야 할 주문은 주문에 있는 남은 음료의 개수가 가장 많은 주문이다. 만약, 남은 음료의 개수가 가장 많은 주문이 여러 개이면 그 중에 가장 먼저 받은 주문이다.

 

Return Value

  RESULT.cnt : 가장 급하게 처리해야 할 주문의 개수

  RESULT.IDs : 가장 급하게 처리해야 할 주문의 ID들
