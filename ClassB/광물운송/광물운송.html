[문제 설명]

크기가 N*N인 격자 모양인 섬에 특수 광물이 발견되어 이 광물을 수집하려고 한다.

이 문제에서는 섬 내의 위치를 (행, 열)로 나타낸다.

 

[Fig. 1]은 N이 10인 섬을 보여준다.

[Fig. 1]

 

섬 내에 이 광물이 발견된 위치마다 베이스캠프를 설치하여 광물을 채굴하게 된다.

베이스캠프는 구분을 위한 ID와 한번에 채굴 가능한 광물의 채굴량이 존재한다.

베이스캠프는 한번에 광물의 채굴량만큼만 채굴할 수 있다.

 

[Fig. 2] (a)는 (3, 4)에 광물이 발견되어 베이스캠프를 설치한 위치를 원 모양으로 보여준다.

[Fig. 2] (b)와 같이 여러 곳에 베이스캠프가 설치될 수 있다.

원 안의 숫자는 채굴량을 의미한다. (3, 4)의 베이스캠프에서는 2만큼 채굴이 가능하다.

[Fig. 2]

 

정부에서는 비행기를 이용하여 이 섬 내의 베이스캠프 중 한 곳에 운송차량을 떨어뜨려 광물을 수집하게 하고,

다시 비행기에 실어 광물 가공 공장에 보내려고 한다.

 

베이스캠프에 떨어진 차량은 섬 내에서 상하좌우로 이동이 가능하다.

차량은 초기에 에너지를 L만큼 가지고 있으며

한 칸 이동시마다 에너지는 1만큼 줄어들고 에너지가 0이 되면 이동을 멈춘다.

베이스캠프에는 에너지 충전소가 있어 차량이 도착하면 차량의 에너지는 다시 L로 충전된다.

 

[Fig. 3]은 에너지 L이 3인 차량의 이동 가능 범위를 노란색으로 보여준다.

 

 

[Fig. 3]

 

[Fig. 3](a)는 (3, 4)에 떨어진 차량의 이동 가능한 범위를 보여준다.

[Fig. 3](b)와 같이 베이스캠프들이 배치되어 있다면 (3, 4)에 떨어진 차량은 (5, 2)에 위치한 베이스캠프로 곧바로 이동할 수 없다.

하지만, (3, 4)에서 (3, 1)로 이동한다면 에너지가 3으로 회복되기 때문에

[Fig. 3](c)에서 보듯이 (5, 2)로 이동이 가능하다.

(3, 4)에 떨어진 차량은 어떤 방법으로도 (8, 6)의 베이스캠프로 이동할 수 없다.

 

차량이 베이스캠프에 도착하면 그 곳의 채굴량만큼 광물들을 수집한다.

차량은 에너지를 가지고 있는 한, 어디로도 이동 가능하며 한번 방문한 베이스캠프에도 다시 방문할 수 있다.

하지만, 그 차량이 수집을 마치고 섬을 떠나기 전까지는 동일한 베이스캠프에서 광물을 한 번을 초과하여 수집할 수 없다.

[Fig. 4]는 광물을 최대한 수집하기 위한 이동경로를 보여준다.

[Fig. 4]

 

[Fig. 4](a)와 같이 베이스캠프가 설치되어 있을 때, (3, 4)에 떨어진 차량의 이동경로를 생각해보자.

[Fig. 4](b)의 이동경로는 차량이 멈추지 않고, 최대로 광물을 수집할 수 있는 방법 중 하나이다.

(3, 4) à (3, 1) à (5, 2) à (3, 1) à (2, 3) à (3, 4) à (4, 6)으로 이동이 가능하다.

한번 방문한 베이스캠프를 다시 방문하여 에너지를 회복할 수 있지만 광물은 한번만 수집되므로,

수집되는 광물들의 총 개수는 18개이며, 수집할 수 있는 광물의 최대값이 된다.

(3, 4)에 떨어진 차량은 L이 3임으로 (8, 6)의 베이스캠프로는 이동이 불가능하다.

 

[Fig. 4](c)에서 빨간 박스 내의 베이스캠프들 중 어느 한 곳에 차량이 떨어져도,

수집할 수 있는 광물들의 총 개수는 18개로 동일한다.

 

[Fig. 5]는 [Fig. 4](a)에서 (6, 6)에 베이스캠프가 추가되는 것을 보여준다.

[Fig. 5]

 

[Fig. 5](a)는 (3, 4)에 L이 3인 차량이 떨어졌을 때 이동가능 경로를 보여준다.

이 때 수집할 수 있는 광물의 최대값은 24개가 된다.

차량이 [Fig. 5](b)의 빨간 박스 내의 어느 베이스캠프에서 출발하여도 수집되는 광물의 최대값은 동일하다.

 

정부에서는 K개 이상의 광물을 수집할 것을 요청할 수 있다.

이 경우, K개 이상의 광물을 수집할 수 있는 베이스캠프의 위치를 찾고 그 곳으로 차량을 떨어뜨려야 한다.

만약, 해당되는 베이스캠프가 2개 이상인 경우, 다음의 우선순위에 따라서 선별해야 한다.

1. 채굴하는 광물의 양이 작은 베이스캠프

2. 1번이 여러 개이면 위치하는 행의 번호가 작은 베이스캠프

3. 2번이 여러 개이면 위치하는 열의 번호가 작은 베이스캠프

 

[Fig. 6] (a)과 같이 베이스캠프가 배치되어 있는 경우를 살펴보자.

[Fig. 6]

 

[Fig. 6] (b)는 에너지 L이 3인 차량이 떨어졌을 때 같은 수량을 수집할 수 있는 베이스캠프들끼리 박스로 묶어놨다.

(8, 1)에 떨어지면 13개를, (3, 4)에 떨어지면 24개를, (0, 8)에 떨어지면 8개의 광물을 수집할 수 있다.

 

정부에서 K값으로 25를 요구했을 때, 25개 이상의 광물을 수집할 수 있는 위치는 없다.

 

K가 10인 경우, 10개 이상의 광물을 수집할 수 있는 2개의 박스 내의 베이스캠프들 중 하나를 선택하면 된다.

즉, (8, 1)이 포함된 박스와 (3, 4)가 포함된 박스 내의 베이스캠프들 중 하나를 선택한다.

우선순위에 따라 광물의 채굴량이 가장 작으면서 행의 번호가 작은 (3, 4)가 선택된다.

 

K가 5인 경우, 5개 이상의 광물을 수집하려면 세개의 박스들에 포함된 베이스캠프들 중 하나를 선택해야 한다.

우선순위가 가장 높은 (0, 8)에 위치한 베이스캠프가 선택된다.

 

 

정부를 도와서 차량을 떨어뜨릴 베이스캠프의 위치를 찾는 프로그램을 작성해야 한다.

 

아래 API 설명을 참조하여 각 함수를 구현하라.

※ 아래 함수 signature는 C/C++에 대한 것으로 다른 언어에 대해서는 제공되는 Main과 User Code를 참고하라.

 

아래는 User Code 부분에 작성해야 하는 API 의 설명이다.

void init(int L, int N)

각 테스트 케이스의 처음에 호출된다.

차량이 이동할 수 있는 에너지가 L로 주어진다.

섬의 크기가 N*N으로 주어진다.

섬의 한 변의 길이 N은 L * 30보다 작거나 같다.

 

초기 섬 내에는 베이스캠프가 없다.

 

__Parameters

____ L : 차량의 이동에너지 (3 ≤ L ≤ 500)

____ N : 섬의 한 변의 길이 (9 ≤ N ≤ L * 30)

int addBaseCamp(int mID, int mRow, int mCol, int mQuantity)

섬에 설치할 베이스캠프의 정보가 주어진다.

베이스캠프의 ID는 mID이고, 위치는 (mRow, mCol)이며, 채굴량은 mQuantity이다.

 

베이스캠프를 설치한 후, 설치된 베이스캠프에서 차량이 출발할 때 수집할 수 있는 광물의 최대값을 반환한다.

 

추가되는 베이스캠프의 ID는 유일함을 보장한다.

베이스캠프가 설치될 위치는 비어있음을 보장한다.

 

__Parameters

____ mID : 설치될 베이스캠프의 ID (1 ≤ mID ≤ 1,000,000,000)

____ mRow : 설치될 베이스캠프의 행번호 (0 ≤ mRow ≤ N-1)

____ mCol : 설치될 베이스캠프의 열번호 (0 ≤ mCol ≤ N-1)

____ mQuantity : 설치될 베이스캠프의 채굴량 (1 ≤ mQuantity ≤ 100)

 

__Returns

_____r베이스캠프를 설치한 후, 베이스캠프에서 차량이 출발할 때 수집할 수 있는 광물의 최대값

int findBaseCampForDropping(int K)

차량을 떨어뜨려 이동시켰을 때 수집할 수 있는 광물의 최대값이 K 이상인 베이스캠프를 찾고,

그 ID를 반환한다.

반환해야 하는 베이스캠프가 2개 이상인 경우

문제 지문에 서술한 가장 큰 우선순위의 베이스캠프의 ID를 반환한다.

존재하지 않는 경우 -1을 반환한다.

 

섬 내에는 베이스캠프가 1개 이상 존재함을 보장한다.

 

__Parameters

____ K : 수집할 광물의 최소량 (1 ≤ K ≤ 2,000,000)

 

__Returns

_____ 차량을 떨어뜨려 K 이상의 광물 수집이 가능한 위치의 베이스캠프의 ID, 없는 경우 -1
