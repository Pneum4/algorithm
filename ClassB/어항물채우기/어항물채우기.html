[문제 설명]

N개의 어항에 구조물을 설치하고, 가장 높게 물을 채울 수 있는 어항을 찾는 시뮬레이션을 진행하려고 한다.

 

동일한 크기의 N개의 어항이 주어진다. 각각의 어항은 서로 다른 번호를 가지고 있다.

각 어항 내에는 다양한 높이의 구조물들이 설치되어 있다.

하나의 구조물은 막대기 모양으로 어항 내의 세로 방향으로 설치된다.

구조물은 방향이 있으며, 위쪽과 아래쪽에 다른 구조물과 결합을 위한 결합판을 가지고 있다.

결합판은 4가지 종류로 {0, 1, 2, 3}으로 주어진다.
이미 설치된 구조물 위에 다른 구조물을 설치하기 위해서는 맞닿는 결합판의 종류가 같아야 한다.

[Fig. 1](a)는 가로 길이가 5이고, 세로 길이가 10인 어항의 모습을 보여준다.
어항에는 열 별로 길이가 {5, 4, 1, 3, 7}인 구조물들이 설치되어 있으며, 구조물들은 위쪽 결합판으로 {1, 2, 1, 0, 1}를 가지고 있다. 초기 설치된 구조물에는 아래쪽 결합판은 없다.

[Fig. 1](b)는 길이가 5이고, 위쪽 결합판은 2, 아래쪽 결합판은 0인 구조물을 보여준다.
이 구조물은 위쪽 결합판이 0인 어항의 4번째 열에 있는 구조물 위에 설치할 수 있고, 설치 후 어항의 모습은 [Fig. 1](c)와 같다.


                                                                       [Fig. 1]

 

구조물이 위와 같은 특성을 가지고 있을 때, 어항에 구조물들을 추가하는 기능을 만들고자 한다.
 

한번에 3개씩 주어지는 구조물들을 동시에 설치해야 한다.

구조물을 설치 할 수 있는 조건은 다음과 같다.

1. 주어지는 3개의 구조물들은 동일한 어항 내에 주어진 순서대로(주어진 배열의 index 순)
   왼쪽에서 오른쪽으로 인접하게 설치되어야 한다.
2. 설치되는 인접한 구조물들은 최소 한 셀의 면이 붙어 있어야 한다.
3. 설치되는 구조물은 어항의 높이보다 높게 설치되면 안된다.

[Fig. 2]는 진한 갈색 구조물 3개가 주어졌을 때, 잘못 설치된 예를 보여준다.


                                                                       [Fig. 2]

[Fig. 2] (a)는 주어지는 구조물들이 서로 인접하지 않게 설치되어 있다.
[Fig. 2] (b)에서는 2번째 열과 3번째 열에 설치된 구조물들이 서로 붙어 있지 않다.
[Fig. 2] (c)에서는 설치된 구조물들 중 하나가 어항의 높이보다 높게 설치되었다.

 

3개의 구조물이 주어지면 N개의 어항들에서 구조물을 설치할 수 있는 위치들을 모두 찾아야 한다.

만약 [Fig. 3]과 같은 어항이 주어져 있는 경우를 살펴보자.


                                                                                          [Fig. 3]

 


                                                                      [Fig. 4]

 

[Fig. 4] (a)와 같이 구조물들이 주어지는 경우, 설치 가능한 위치는 총 3개이다.
[Fig. 5]의 빨간 색 박스 위에 설치 가능하다.

하지만 [Fig. 4] (b)와 같이 구조물들이 주어지는 경우, 설치 가능한 위치는 총 2개이다.
[Fig. 5]에서 ID가 30인 어항에 설치하면 구조물들 중 하나가 어항 높이보다 높게 설치됨으로 이 위치에는 설치할 수 없다.

[Fig. 4] (c)와 같이 구조물들이 주어지는 경우, 설치 가능한 위치는 총 2개이다.
[Fig. 5]에서 ID가 40인 어항의 파란색 박스들 위에만 설치가 가능하다.
첫번째 열에는 구조물들이 붙어있을 수 없기 때문에 설치할 수 없다.


                                                                                          [Fig. 5]

 

위와 같이 찾아낸 위치들 중 우선순위가 가장 높은 곳에 실제로 설치를 할 수도 있다.

구조물들의 설치를 위한 우선순위는 아래와 같이 판단한다.

1. 어항의 ID가 작을수록 우선순위가 높다.
2. 같은 어항내에서는 왼쪽으로 갈수록 우선순위가 더 높다.

 

 

구조물이 설치된 어항들에 주어진 물을 쏟아서 가장 높게 물이 차오르는 어항을 찾는 기능도 구현해야 한다.

 

물을 어항에 쏟아 부었을 때, 구조물들은 모두 작은 구멍들이 뚫려 있어 물이 다른 열로 이동할 수 있다.
그리고, 구멍의 크기는 무척 작기 때문에 구멍 속에 들어가는 물의 양은 무시할 수 있다.

 

물의 양 1은 어항 내의 하나의 셀을 채울 수 있는 양으로, 여러 개의 셀에 나눠서 채울 수는 없다.

 

어항에 물을 채울 때는 다음과 같은 방법으로 채워야 한다.

1. 물은 최소 1이상을 사용해서 채워야 한다.
2. 주어진 물의 양 내에서 물이 포함된 행에 빈 공간이 없도록 물을 채워야 한다.
3. 주어진 물의 양 내에서 가장 높게 물을 채워야 한다.
4. 물은 어항의 높이보다 높게 채울 수 없다.

 

[Fig. 6]은 물을 채울 때 어항의 모습을 보여준다. 파란색 셀은 물을 의미한다.


                                                                       [Fig. 6]

[Fig.6] (a)에서 주어지는 물의 양이 3인 경우, 물을 1이상을 사용해야 하는 조건을 만족할 수 없음으로 이 어항에는 물을 채울 수 없다.

[Fig.6] (b)에서 주어지는 물의 양이 3인 경우, 물을 채울 수 있는 최대 높이는 3이 된다.

[Fig.6] (b)에서 주어지는 물의 양이 5가 되더라도, 4번 째 행에 빈공간이 없도록 물을 채울 수 없기 때문에 채울 수 있는 최대 높이는 3이 된다.

[Fig.6] (c)에서 주어지는 물의 양이 13인 경우 물을 채울 수 있는 최대 높이는 5가 되고, 이때 사용한 물의 양은 11이 된다.

 

물의 양이 주어지면, N개의 어항 중 물을 쏟아서 가장 높게 채울 수 있는 어항을 찾아야 한다.

채울 수 있는 높이가 같은 어항들이 여러 개인 경우 우선 순위가 가장 높은 어항을 선택한다.

1. 해당 높이까지 채우기 위해 사용하는 물의 양이 많을수록 우선순위가 높다.

2. 같은 물의 양을 사용한다면 어항의 ID값이 작을수록 우선순위가 높다.

 

N개의 어항 정보를 주었을 때, 주어지는 구조물들을 설치할 수 있는 위치들을 찾고, 주어지는 물의 양을 이용하여 최대 높이까지 물을 채울 수 있는 어항을 찾는 프로그램을 작성해야 한다.

 

아래 API 설명을 참조하여 각 함수를 구현하라.

※ 아래 함수 signature는 C/C++에 대한 것으로 다른 언어에 대해서는 제공되는 Main과 User Code를 참고하라.

 

아래는 User Code 부분에 작성해야 하는 API 의 설명이다.

void init(int N, int mWidth, int mHeight,
                     int mIDs[], int mLengths[][500], int mUpShapes[][500])

각 테스트 케이스의 처음에 호출된다.
가로길이 mWidth이고, 세로길이 mHeight인 어항 N개의 정보가 주어진다.

i번 째 어항의 ID는 mIDs[i]이고, 어항 내에 설치된 구조물 mWidth개의 정보가
mLengths[i]와 mUpShapes[i]로 주어진다.

각 열에 설치된 구조물들의 길이는 배열 mLengths[i]로 주어지며,
위쪽 결합판의 종류들은 배열 mUpShapes[i]로 주어진다. (0 ≤ i ≤ N-1)

mIDs내의 값들은 모두 다름을 보장한다.

모든 어항에는 물이 들어있지 않다.

 

__Parameters

_____N : 어항의 개수 (5 ≤ N ≤ 20)

_____mWidth : 어항의 가로 길이 (5 ≤ mWidth ≤ 500)
_____mHeight : 어항의 세로 길이 (10 ≤ mHeight ≤ 1,500)
_____mIDs[] : 어항의 ID (1 ≤ mIDs[] ≤ 1,000,000)
____ mLengths[][] : 각 어항 별, 열 별 설치된 구조물의 높이 (1 ≤ mLengths[][] ≤ mHeight)
____ mUpShapes[][] : 각 어항 별, 열 별 설치된 구조물의 위쪽 결합판의 종류 (0 ≤ mUpShapes[][] ≤ 3)

int checkStructures(int mLengths[],
                            int mUpShapes[], int mDownShapes[])

주어진 구조물들을 설치할 수 있는 위치들의 수를 반환한다.

 

설치할 구조물은 3개가 주어지며, i번째 구조물의 길이는 mLengths[i], 구조물의 위쪽 결합판의 종류는 mUpShapes[i], 아래쪽 결합판의 종류는 mDownShapes[i]로 주어진다. (0 ≤ i ≤ 2)

 

우선순위를 고려하지 않고 주어진 구조물들을 설치 가능한 위치를 모두 찾아야 한다.

 

설치할 수 있는 위치가 하나도 없는 경우 0을 반환한다.

 

__Parameters

____ mLengths[] : 구조물의 높이 (1 ≤ mLengths[] ≤ mHeight / 2)
____ mUpShapes[] : 구조물의 위쪽 결합판의 종류 (0 ≤ mUpShapes[] ≤ 3)

____ mDownShapes[] : 구조물의 아래쪽 결합판의 종류 (0 ≤ mDownShapes[] ≤ 3)

 

__Returns

_____주어진 구조물들을 설치할 수 있는 모든 위치의 수

int addStructures(int mLengths[],
                            int mUpShapes[], int mDownShapes[])

주어진 구조물들을 설치하고, 설치한 어항의 ID와 설치한 열 위치를 반환한다.

반환하는 값은 “어항의 ID * 1000 + 열 위치” 이다.

설치할 구조물들의 개수는 3개이며, i번째 구조물의 길이는 mLengths[i], 구조물의 위쪽 결합판의 종류는 mUpShapes[i], 아래쪽 결합판의 종류는 mDownShapes[i]로 주어진다. (0 ≤ i ≤ 2)

 

구조물들의 설치 가능한 위치와 우선순위는 지문을 참고하라.

 

설치할 수 있는 어항이 없는 경우 0을 반환한다.

 

__Parameters

____ mLengths[] : 구조물의 높이 (1 ≤ mLengths[] ≤ mHeight / 2)
____ mUpShapes[] : 구조물의 위쪽 결합판의 종류 (0 ≤ mUpShapes[] ≤ 3)

____ mDownShapes[] : 구조물의 아래쪽 결합판의 종류 (0 ≤ mDownShapes[] ≤ 3)

 

__Returns

_____주어진 구조물들을 설치하는 어항과 위치의 정보, 설치할 수 있는 어항이 없는 경우 0을 반환한다.

Result pourIn(int mWater)

주어지는 물의 양은 mWater이며, 주어진 물을 이용해서 가장 높게 채울 수 있는
어항 중 우선순위가 가장 높은 어항의 정보를 반환한다.

Result.ID : 반환할 어항의 ID

Result.height : 반환할 어항의 물로 채울 수 있는 최대 높이

Result.used : 반환할 어항의 가장 높게 채우기 위해 사용되는 물의 양

 

물을 채울 수 있는 어항이 없는 경우 Result.ID를 0으로 반환한다.

 

물을 채우는 방법과 우선순위는 지문을 참고하라.

 

실제로 물을 채우지는 않음에 유의하라.

즉, 함수 호출 후 모든 어항에는 물이 존재하지 않는다.

 

__Parameters

     mWater : 주어지는 물의 양 (1 ≤ mWater ≤ 3,000)

 

__Returns

[제약사항]

1. 각 테스트 케이스 시작 시 init() 함수가 호출된다.

2. 각 테스트 케이스에서 checkStructures() 함수는 최대 10,000 회 호출된다.

3. 각 테스트 케이스에서 addStructures() 함수는 최대 1,000 회 호출된다.

4. 각 테스트 케이스에서 pourIn() 함수는 최대 100 회 호출된다.

5. 함수 호출시 주어지는 3개의 구조물들을 설치할 수 있는 위치의 개수는
  구조물을 설치 할 수 있는 조건 2, 3번을 제외할 경우 최대 500개이하임을 보장한다.

 

[입출력]

입출력은 제공되는 Main 부분의 코드에서 처리하므로 User Code 부분의 코드에서는 별도로 입출력을 처리하지 않는다.

Sample input 에 대한 정답 출력 결과는 “#TC번호 결과” 의 포맷으로 보여지며 결과가 100 일 경우 정답, 0 일 경우 오답을 의미한다.
