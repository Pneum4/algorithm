[문제 설명]

아래 [Fig. 1]과 같이 N x N 크기의 정사각형 모양의 독가스로 가득 찬 던전이 있다.

                                           [Fig. 1]

 

던전은 길(흰색)과 기둥(회색) 그리고 게이트(①모양)로 이루어져 있다.
그러나 게이트는 기사가 이동중이 아닐 때 임의의 길 위에 생성되기도 하고 사라지기도 한다.

 

기사는 열려 있는 두 개의 게이트 중 어느 한 게이트에 들어와서 다른 게이트로 탈출해야 한다.
나머지 게이트들은 모두 닫혀있다.
기사는 길과 게이트로는 상하좌우로 1칸씩 이동할 수 있고 기둥으로는 이동할 수 없다.

 

초기에 기사가 게이트로 들어올 때는 mMaxStamina의 체력을 가진다.
그리고 기사가 1칸씩 움직일 때 체력이 1씩 감소하고 시간은 1씩 증가한다.
만약 기사의 체력이 0이 되면 더 이상 움직일 수 없다.
하지만, 도중에 임의의 닫혀있는 게이트를 만나면 기사는 맑은 공기를 마실 수 있어서 체력이 mMaxStamina가 된다.

 

열려 있는 입장 게이트와 탈출 게이트가 주어질 때 기사는 가능한 최소 시간에 던전을 탈출하고 싶어한다.

 

예를 들어 보자.

12x12 크기의 던전을 체력이 6인 기사가 게이트 2로 던전에 들어와 게이트 3으로 탈출한다고 가정해보자.
아래 [Fig. 2]와 같이 게이트 5를 거쳐 이동하면 체력이 회복되어 최단 시간에 던전을 탈출할 수 있다.
이 때 경로가 던전을 탈출할 수 있는 최단 시간이므로 최소 시간은 12가 된다.

                                           [Fig. 2]

 

 그리고 역시 체력이 6인 기사가 게이트 4로 던전에 들어와 게이트 3으로 탈출한다고 했을 때 아래 [Fig. 3]과 같이 이동하면 던전을 탈출할 수 없다.

                                           [Fig. 3]

 

하지만 [Fig. 4]와 같이 이동하는 경우는 던전을 탈출할 수 있게 된다.
이 때 경로가 던전을 탈출할 수 있는 최단 시간이므로 최소 시간은 17이 된다.

                                           [Fig. 4]

 

그러나 체력이 6인 기사가 게이트 5로 들어와 게이트 6으로 탈출하려 한다면 현재 조건으로는 던전을 탈출할 수 없다.

 

기사를 도와 독가스로 가득 찬 던전을 최단 시간에 탈출하는 프로그램을 만들어 보자.

 

아래 API 설명을 참조하여 각 함수를 구현하라.

※ 아래 함수 signature는 C/C++에 대한 것으로 다른 언어에 대해서는 제공되는 Main과 User Code를 참고하라.

 

아래는 User Code 부분에 작성해야 하는 API 의 설명이다.

void init(int N, int mMaxStamina, int mMap[][])

각 테스트 케이스의 처음에 호출된다.
N는 던전 지도의 크기이며, 기사의 최대 체력으로 mMaxStamina 가 주어진다.
기사는 mMaxStamina로 입력 받은 체력만큼만 던전을 이동할 수 있다.
그리고 mMap은 길(0)과 기둥(1) 정보가 포함된 던전 지도이다.

 

아래 [Fig. 5]는 N이 12인 던전의 지도의 입력을 보여준다.
i번째 행, j번째 열에 있는 지도의 정보는 mMap[i][j]로 나타내며 그 값이 0일 경우 길을, 1일 경우 기둥을 의미한다. (행과 열은 0부터 시작한다.)

[Fig. 5]

 

Parameters

  N : 던전 지도의 크기 ( 12 ≤ N ≤ 350 )

  mMaxStamina : 기사의 최대 체력(이동 가능 거리) ( 5 ≤ mMaxStamina ≤ 100 )

  mMap : 던전의 지형 정보( N x N ) ( 0 ≤ mMap[][] ≤ 1 )

void addGate(int mGateID, int mRow, int mCol)

ID값이 mGateID인 게이트를 지도 상에 mRow번째 행, mCol번째 열인 (mRow, mCol) 위치에 추가한다.
mGateID는 게이트의 ID이며, mRow와 mCol는 게이트가 추가될 던전 지도상의 위치이다.
mGateID는 처음 호출될 때 1 이고, 이 후 1씩 증가한다.

 

(mRow, mCol) 위치에는 게이트나 기둥이 없음을 보장한다.

 

Parameters

  mGateID : 게이트 ID ( 1 ≤ mGateID ≤ 200 )

  mRow : 던전 지도의 행 좌표 ( 0 ≤ mRow ≤ N - 1 )

  mCol : 던전 지도의 열 좌표 ( 0 ≤ mCol ≤ N - 1 )

void removeGate(int mGateID)

ID값이 mGateID인 게이트를 제거한다.
mGateID는 게이트의 ID이다.

 

mGateID는 addGate() 함수로 추가된 게이트 ID만 주어지며, 삭제되거나 존재하지 않는 게이트 ID 값은 입력으로 주어지지 않음을 보장한다.

 

Parameters

  mGateID : 게이트 ID ( 1 ≤ mGateID ≤ 200 )

int getMinTime(int mStartGateID, int mEndGateID)

기사는 mStartGateID번 게이트에서 출발하여 mEndGateID번 게이트로 최단 시간 내에 도착할 수 있는 방법을 찾고 그 시간을 반환해야 한다.
만약 mEndGateID번 게이트에 도착할 수 있는 방법이 없다면 -1을 반환한다.
기사는 출발 시에 최대 체력인 mMaxStamina 값을 가지고 있다.

 

mStartGateID번 게이트와 mEndGateID번 게이트는 서로 다르고, 던전 내에 존재함을 보장한다.

 

Parameters

  mStartGateID : 시작 게이트 ID ( 1 ≤ mStartGateID ≤ 200 )

  mEndGateID : 종료 게이트 ID ( 1 ≤ mEndGateID ≤ 200 )

 

Returns

  시작 게이트에서 종료 게이트까지 이동한 최단 시간을 반환한다.

  던전을 탈출할 수 없을 경우 -1을 반환한다.
