[문제 설명]

N명의 선수들이 경기를 진행하려고 한다.

선수들의 수가 많기 때문에 여러 개의 리그로 나눠서 경기를 진행한다.
리그의 개수는 L로 주어지며, 각각 0부터 L-1까지의 ID 값을 가진다.
리그의 ID값이 작을수록 더 우수한 리그임을 의미한다.

N명의 선수들은 각각 0부터 N-1까지의 ID 값을 가지고 있고 각각 능력 값을 가지고 있다.

선수의 능력은 능력 값과 ID 값으로 평가된다.
능력 값이 높을수록 능력이 좋은 선수이다. 능력 값이 같다면 ID가 작을수록 더 능력이 좋은 선수임을 의미한다.
 

초기에는 앞 번호 리그부터 선수들의 ID 순서대로 N/L 명씩 차례대로 배치한다.
이때, N은 L의 배수이며, N/L은 홀수 값을 갖는다.

 

예를 들어 12명의 선수들에 대한 ID 값과 능력 값이 아래와 같이 주어지며,
4개의 리그가 운영되는 경우를 생각해보자.


                                                               [Fig. 1]

 

4개의 리그에 각각 3명씩 순서대로 배치하면 아래와 같다.


                                                              [Fig. 2]

 

리그 승강제를 도입하여, 각각의 리그에서 능력이 가장 좋지 않은 선수는 바로 아래 리그로 내려가고,
리그에서 능력이 가장 좋은 선수는 바로 위 리그로 올라간다.
(리그의 ID가 i라고 한다면 바로 위 리그의 ID는 i-1이고, 바로 아래 리그의 ID는 i+1이 된다.)

모든 리그에서 이동을 하기 때문에 L개의 리그가 있다면 이동하는 선수들은 총 (L-1)*2명이 된다.
0번 리그의 능력이 가장 좋은 선수와 L-1번 리그의 능력이 가장 좋지 않은 선수는 이동하지 않는다.

[Fig. 3]은 [Fig. 2]에서 리그 별 선수들을 높은 능력 순으로 정렬한 상태를 보여준다.
1번 리그에서 능력이 가장 좋은 선수는 5번 선수이고 가장 좋지 않은 선수는 3번 선수이다.


                                                              [Fig. 3]

 

승강제로 인하여 선수들의 이동이 있은 후 리그들 내의 모습은 [Fig. 4]와 같다.
이동한 선수들은 0, 5, 3, 8, 6 그리고 11번 선수들이고 이들 ID 값들의 합은 33이다


                                                               [Fig. 4]
 

승강제 만으로는 하위 리그들에 능력이 좋은 선수를 배치시킬 수 없기 때문에 트레이드 제도도 운영된다.
트레이드 제도란, 각각의 리그에서 능력이 가장 좋은 선수를 바로 위 리그의 중간 급 능력의 선수와 맞교환하는 제도를 의미한다.
리그 내의 중간 급 능력의 선수란 리그 내에 M명이 존재할 때, ((M+1) / 2)번째 능력이 좋은 선수를 의미한다.
(가장 능력이 좋은 선수는 첫번째로 능력이 좋은 선수이다.)
 

예를 들어 리그 내에 9명의 선수가 있는 경우 (9+1) / 2 = 5번째 능력이 좋은 선수가 중간 급 선수가 된다.
0번 리그의 능력이 가장 좋은 선수와 L-1번 리그의 중간급 능력의 선수는 이동하지 않는다.

 

[Fig. 5]은 [Fig. 4]에서 리그별 선수들을 높은 능력 순으로 정렬한 상태를 보여준다.

                                                               [Fig. 5]

 

선수들 간의 트레이드를 한 후의 리그들 내의 상태는 [Fig. 6]과 같다.
리그내 3명의 선수들이 있으므로 중간급 능력의 선수는 2번째로 능력이 좋은 선수가 된다.
1번 리그에서는 8번 선수가 중간급 능력의 선수가 된다.
이동한 선수들은 2, 4, 8, 11, 3 그리고 6번 선수들이며, 이들 ID 값들의 합은 34이다


                                                              [Fig. 6]

 

위와 같이 운영되는 리그 제도를 시뮬레이션하는 프로그램을 작성하려고 한다.

 

아래 API 설명을 참조하여 각 함수를 구현하라.

※ 아래 함수 signature는 C/C++에 대한 것으로 다른 언어에 대해서는 제공되는 Main과 User Code를 참고하라.

 

아래는 User Code 부분에 작성해야 하는 API 의 설명이다.

void init(int N, int L, int mAbility[])

각 테스트 케이스의 처음에 호출된다.
N명의 선수들이 L개의 리그에 참여하게 된다.
선수의 ID는 0부터 N-1까지 순차적으로 부여되고, ID가 i인 선수의 능력 값은 mAbility[i]이다.
리그의 ID는 0부터 L-1까지 순차적으로 부여된다.
앞 번호 리그부터 선수들의 ID 순서대로 N/L 명씩 차례대로 배치한다.
N은 L의 배수이며, N / L은 홀수이다.

 

__Parameters

_____N : 선수들의 수 (9 ≤ N ≤ 39,990)

_____L : 리그의 개수 (3 ≤ L ≤ 10, 3 ≤ N / L ≤ 3,999)

_____mAbility : 각 선수들의 능력 값 (1 ≤ mAbility[] ≤ 10,000)

int move()

각 리그 별로 능력이 가장 좋은 선수와 상위 리그의 가장 나쁜 선수를 선발하여 교환하고,
전체 리그에서 이동한 선수들의 ID 값의 합을 반환한다.

능력의 비교는 위의 지문을 참고하라.
 

__Returns

_____이동한 선수들의 ID 값의 합

int trade()

각 리그 별로 능력이 가장 좋은 선수와 상위 리그의 중간급 선수를 선발하여 교환하고,

전체 리그에서 이동한 선수들의 ID 값의 합을 반환한다.
능력의 비교는 위의 지문을 참고하라.

 

__Returns

_____ 이동한 선수들의 ID 값의 합
